SSH的配置

SSH的配置

安装 ssh

$sudo apt-get update  #更新软件源
$sudo apt-get install ssh #安装

修改配置文件

OpenSSH（即常说的 ssh）的常用配置文件有 ssh_config 和 sshd_config 。其中 ssh_config 为客户端配置文件，设置与客户端相关的应用可通过此文件实现；sshd_config 为服务器配置文件，设置与服务器相关的应用可通过此文件实现。

一般来说我们常用的都是 sshd_config 配置文件，在 sshd_config 配置文件中，以“# ”（#加空格）开头的是注释信息；以“#”开头的是默认配置信息。

vi /etc/ssh/sshd_config

端口

port 22

设置 sshd 监听端口号，默认情况下为 22，可以设置多个监听端口号，即重复使用 Port 这个设置项。修改后记得重启 sshd ，以及在防火墙中添加放行端口。出于安全考虑，建议端口指定为小于等于 65535，并且非 22 的值。

监听地址

ListenAddress 0.0.0.0 //监视所有IPV4地址
ListenAddress :: //可以自行设置，默认仅使用IPV4

设置 sshd 监听（绑定）的 IP 地址，0.0.0.0 表示监听所有 IPv4 的地址。出于安全考虑，设置为指定 IP 地址，而非所有地址。:: 是IPv6 的地址不需要改。

协议版本

Protocol 2

有部分 sshd_config 配置文件会有此选项。这个选项设置的是 ssh 协议版本，可以是 1 也可以是 2。出于安全考虑，设置为最新协议版本。

密钥

HostKey /etc/ssh/ssh_host_rsa_key //rsa密钥
HostKey /etc/ssh/ssh_host_dsa_key //dsa密钥
HostKey /etc/ssh/ssh_host_ecdsa_key //ecdsa密钥
HostKey /etc/ssh/ssh_host_ed25519_key //ed25519密钥

其对应了四种不同加密类型的密钥，按照个人需求，启用以后记得将密钥放入对应的文件夹内，同时客户端也需要导入密钥才能连接。

登录日志

SyslogFacility AUTH
SyslogFacility AUTHPRIV

当有人使用 ssh 登录系统时，ssh 会记录信息，记录类型为 AUTHPRIV，sshd 服务日志存放在 /var/log/secure。

sshd 服务日志

LogLevel INFO

设置记录 sshd 日志信息的级别。

连接超时

LoginGraceTime 2m

不带单位就默认算作秒，超时了 ssh 连接强制下线。

允许Root用户登录

PermitRootLogin yes

虽然开启了不安全，不过我就是喜欢以 root 用户登录。

密码验证

PasswordAuthentication yes

是否使用密码验证。当然也可以设置为 no，这样就不会使用密码验证，转而使用密钥登录。

空密码登录

PermitEmptyPasswords no

是否允许空密码的用户登录，默认为 no，不允许。

登陆提示

PrintMotd yes

打印登录提示信息，提示信息存储在 /etc/moed 文件中。

上次登录信息

PrintLastLog yes

显示上次登录信息。

反向解析

UseDNS yes

是否使用 DNS 反向解析，默认这条会被注释掉。

PAM 认证

UsePAM yes

是否启用PAM插件式认证模块。

记得保存哦。先 [ESC] 进入命令行模式，再 :wq 保存。

开机自启

systemctl enable ssh
