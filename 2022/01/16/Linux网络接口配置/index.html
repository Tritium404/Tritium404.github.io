Linux网络接口配置

Linux 网络接口配置

对于 Debian 系

debian 的网卡配置文件在 /etc/network/interfaces，我们 cat 一下文件内容：

# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

source /etc/network/interfaces.d/*

# The loopback network interface
auto lo
iface lo inet loopback
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           
# The primary network interface
allow-hotplug ens33
iface ens33 inet static
        address 192.168.1.191/24
        gateway 192.168.1.1
        # dns-* options are implemented by the resolvconf package, if installed
        dns-nameservers 119.29.29.29
        dns-search debian_1

环回接口

开头的注释就是说这个文件描述了可用的网络接口。接下来是环回网络接口：

# The loopback network interface
auto lo
iface lo inet loopback

第二行是 lo(op) 接口是在系统启动时会被自动配置的，第三行是将 lo 接口设置为一个本地回环 (loopback) 地址，在 IPV4 下 ping 127.0.0.1 就可以有回应。

# The primary network interface
allow-hotplug ens33
iface ens33 inet static
        address 192.168.1.191/24
        gateway 192.168.1.1
        # dns-* options are implemented by the resolvconf package, if installed
        dns-nameservers 119.29.29.29
        dns-search debian_1

网卡接口

检测到热插拔才启动 ens33 接口

注意 allow-hotplug 就是允许热插拔。

hotplug 与 auto 的区别是，前者检测到热插拔事件（硬件已接入）才启动接口；后者是自动无论接入否都会启动。

VMware 等虚拟机会根据物理机的网卡特性提供不同的虚拟网卡，所以我在同一台物理机上装不同的发行版是网卡的名称是相同的（但是也有的发行版对网卡的命名规则是不同的）。

第二行开始 iface 配置 ens33 接口，静态地址。设置 ip 地址，子网掩码 24 位。然后是网关。下面是 DNS 选项由 resolvconf 包提供（如果已安装的话）。

我们来列个表格，这样详细一点，iface 参数构成：

iface [接口] [协议] [分配方式]
参数名 可选值 备注
接口 由系统提供 en 标识 ethernet（以太网）；o 标识主板板载网卡；p 标识独立网卡，PCI 网卡；s 标识热插拔网卡，USB 之类的扩展槽引导号；数字标识 MAC 地址与主板信息计算得出的唯一序列。
IP 协议版本 inet/inet6 inet 是 internet 的缩写。inet 指代 ipv4 ； inet6 指代 ipv6。
地址分配方式 static/dhcp/ manual 静态地址与动态分配。最后者没有为接口定义 IP 地址。通常由作为桥接或聚合成员的接口，需要以混杂模式运行的接口（例如，端口镜像或网络 TAP）或在其上配置了 VLAN 设备的接口使用。这是保持接口不带 IP 地址的一种方法。
选项 值 描述
address ipv4 或ipv6 地址/子网掩码 子网掩码可选，指定主机号位数。
gateway ipv4/ipv6 地址 网关，指定局域网出口，用于与外界通信。
dns-nameservers ipv4/ipv6 地址 DNS 解析服务器地址，可以指定给路由器，也可以指定外部地址。需要 resolvconf 包的支持。如果有备用 DNS 的话，请这样书写：dns-nameservers 12.34.56.78 12.34.56.79。
dns-search 主机名 域名解析服务器地址。dns_search 它的一个用处是程序只须要知道主机名就能够解析到 ip，没必要知道域后缀是什么。作为域附加到主机查询中，以创建FQDN。详情：dns配置文件 /etc/resolv.conf中search设置详解_云淡风轻-CSDN博客_dns search
netmask 子网掩码 点分十进制，在 ipv6 则不需要，因为 ipv6 已经没有子网掩码的概念了。在 ipv6 中前 48 位表示网络前缀，其后的 16 位是子网 ID 最后 64 位是接口标识。
vlan-raw-device 接口 在 VLAN 接口上，定义其“父亲”。
bridge_ports 接口 在网桥接口上，定义其成员。
down 控制命令 使用以下命令代替来关闭接口 ifdown。
post-down 控制命令 接口关闭后立即采取的措施。
pre-up 控制命令 接口启动之前的操作。
up 控制命令 使用以下命令代替接口 ifup。使用上的任何选项完全取决于您的想象 iputils。例如，我们可以在操作过程中 up ip link set $IFACE up mtu 9000 启用巨型帧 ( Jumbo_frame )，up（而不是使用 mtu 选项本身）。您也可以调用其他任何软件，例如 up sleep 5; mii-tool -F 100baseTx-FD $IFACE 在接口启动后 5 秒钟强制 100Mbps 全双工。
hwaddress ether 48 位冒号分隔 更改接口的 MAC 地址，而不要使用硬编码到 ROM 中或由算法生成的地址。您可以使用关键字 random 来获取随机的 MAC 地址。
wpa-ssid Unicode字符 服务区标识（名称标识）。
mtu 0-1942 MTU 大小。mtu 9000=巨型帧。如果您的 Linux 主机与支持与更大 MTU 大小的交换机连接，则很有用。当然，这可能会破坏某些协议。
wpa-psk 8到63个 ASCII 字符，或64 个 16 进制数字 十六进制编码的 PSK。
ip_rp_filter 0/1/2 反向过滤（检查）。0 为关闭 [不管数据的进来的接口 与根据路由所决定的返回接口是否一致，也不管返回数据的目标地址（进来数据时的源地址）根据 route 是否可以可达。]。1 表示松散限制，只要返回数据的目标地址（进来数据时的源地址）根据route可以可达（而不管数据进来的接口是否就是返回数据出去的接口），就可以通过。2 严格限制 ，只要数据的进来的接口 与根据路由所决定的返回接口不一致数据就不允许通过。
