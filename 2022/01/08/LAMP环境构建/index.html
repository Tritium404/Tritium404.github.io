LAMP 环境构建

LAMP 环境构建

何为 LAMP

LAMP 是四个软件的集合 (Linux-Apache-MySQL-PHP)，也是目前国际流行的Web应用框架，其中包括Linux操作系统、Apache Web 服务器、MySQL/MariaDB 数据库和 PHP 编程语言环境以及相关组件支持。其实也不一定是以上这四个软件，还有得选。

众所周知，Linux 上的软件包门类繁多，有分商业版和开源版等，而开源又能分出开源协议不同的发行版本。以 MySQL 为例，最早由一家瑞典公司开发，后面这家公司被 Sun 公司收购，Oracle 又把 Sun 收购了，所以 MySQL 的所有权归 Oracle 所有了。在两次收购中离开的工程师们就创立了它的新分支 MariaDB，这玩意儿大致是兼容 MySQL，不过我们在部署服务器的时候，没钱就尽量选择开源的，万一哪天开源版本不维护了呢？比如 Oracle 对待开源软件的态度简直比白雪公主的后妈还毒，所以许多发行版的默认软件仓库已经剔除了 MySQL。

我们用一个表格列出四个软件的详细介绍，同时提供一些可替代的选项（注意，开源几乎代表了免费）：

软件名称 描述 可替代选项（不代表优劣） 描述
操作系统 Linux 老朋友了，不多说了。发行版太多太多，这里都是拿 Debian 举例的。 FreeBSD 一种开源的类 Unix系统，实际上它是兼容 Unix 的，不过因为法律纠纷发就只能称作为类 Unix 系统。对了，这家伙也很稳定，也是用来搭建服务器的不二选择。
HTTP 服务 Apache 开源。全名叫做 Apache HTTP Server Project，Apache 只是 Apache 基金会的一个项目。Apache Web服务器可以添加多功能模块，比如 IPV6，FTP 等等。比起右边那位的优点是模块多，稳定性好。 Nginx 开源。作 HTTP 服务器是它的用途之一，它也可以作为邮件代理服务器、通用的 TCP 代理服务器。比起左边那位，它内存占用小，资源消耗少，同时抗并发，nginx 处理请求是异步非阻塞的，而apache 则是阻塞型的，在高并发下nginx 能保持低资源低消耗高性能（并发可以简单的理解为多任务能力）。
数据库 MySQL MySQL 原本是开源的，但是现在 MySQL 的开源许可证被分为免费的社区版与收费的标准版、企业版等。 MariaDB/Percona/PostgreSQL MariaDB 是左边的分支，大概兼容左边那位，特性上有些不一样。Percona 也是左边那位的分支，在高负载、高并发情况下表现非常突出，乃至阿里巴巴 OceanBase 数据库都要借 Percona Server。PostgreSQL 最近挺火，据说很好用。
脚本语言 PHP 据说 PHP 是最好的的语言？双重意义上的。一是它运行很快，二这是玩梗，自己查。 Python 脚本语言这玩意儿得看需求，上面要是搭的 Wordpress 那就必须的 PHP 老兄上马了。对了 Python 优秀在易用性上。

开始搭建

小爷我今日开辟虚拟机一台，只予核心一颗，内存 2GB，足矣（毕竟我真机是 Intel Core i5-10400，一个核心还没有这么拉跨，虽然放服务器里面算拉跨的了），对了模拟的是 SSH 连接下的操作（我用的 Finalshell），以下全部操作基于 Root 账户。

对于 Debian 系（如 Ubuntu 等）

Apache 的安装

Debian 采用了 apt 的包管理组件，故我们使用 apt-get 命令：

apt-get update #更新下软件包列表
apt-get install apache2 #安装apache2

sudo 允许非 su(superUser) 执行符合安全策略下的需要 su 权限的命令，如果你使用的其他账户登录，请在命令前加上 sudo

会出现一个提问，意思是安装这玩意儿要消耗多少存储空间，你希望继续吗？此时我们输入 y 回车确认就行（这是每次安装软件都有的提问，以后将不再特意点出）。

如果有必要的话，你也可以确认下是否安装成功：

apt list --installed | grep apache2

此操作将列出所有安装的软件并使用 grep 将列出的软件包的输出进行关键字的匹配，结果如下：

apache2-bin/stable,stable-security,now 2.4.51-1~deb11u1 amd64 [已安装，自动]
apache2-data/stable,stable-security,now 2.4.51-1~deb11u1 all [已安装，自动]
apache2-utils/stable,stable-security,now 2.4.51-1~deb11u1 amd64 [已安装，自动]
apache2/stable,stable-security,now 2.4.51-1~deb11u1 amd64 [已安装]

那你肯定会问我：俺只装了一个 apache2 ，这里咋又多蹦出三个呢？

事实上你可以把前三个软件理解为构成 apache2 的组件，它们分别是二进制文件，数据和工具。

apt install mariadb-server

太对了，正在运行。没有启动的话就手动启动，不用我再教了。

mysql_secure_installation

启用安全安装。它会让你注意下面的步骤：“嘿，我的老伙计，为了你的数据库安全，请小心一点！”，然后填写你 root 账户的密码。

在 Linux 中，只要是输入密码的场合，你怎么按光标都是不会移动的，不过内容实际上是输入进去的，这点请放心。

然后它就会问你是不是要设置 unix_socket 认证，如果你的 root 账户设置了密码倒是可以不用设置。我们安全地回答 n 。

然后它又问你需要更改 root 密码吗？我们回答 n 。

然后又问你需要移除匿名用户吗？那是当然，我们回答 y 。

然后问你需要禁止远程 root 用户登陆吗？禁用了一点不方便，当然回答 n。

够了够了已经很多个然后了，移除测试数据库吗？移除 y。

问你保存设置并且立马载入吗？回答y。

好了你的数据库应该挺安全了。

咱接下来输入：

mysql -u root -p

以 root 用户登录 MySQL。

进入数据库成功然后就根据你的上层需求来配置数据库了。

配置 PHP 环境

apt-get update #更新下软件包列表
sudo apt install php libapache2-mod-php php-mysql #装三个软件

有的人可能要问，我明明要装的是 php 啊，为什么还要装那那个额外的软件？

libapache2-mod-php 的作用是让 apache 能够解析 php，而 php-mysql 的作用是能够让 php 对 mysql 操作。

安装完以后，我们再来测试下 php 的关联正不正常，apache 能不能解析到 php 文件。

总所周知，我搭 LAMP 环境那就是为了建网站，那么那些网页放哪里呢？也包括我们待会要用到的 phpinfo 测试页。

我们可以把他们存储到 /var/www/html 下，于是乎，我们在其下创建一个 php 文件：

cd /var/www/html #载入到该目录
touch info.php #新建该文件
vi info.php

然后按 i 进入编辑模式，键入以下内容：

<?php
phpinfo();

然后一如既往地 :wq 保存。保存好以后，我们来访问测试页：http://127.0.0.1/info.php

如果你是用的桥接模式，请直接访问：http://你的虚拟机在内网的ip/info.php

如果出现了一个紫色的 php 测试页那就是成功了。不用在意上面的文字，你只要出现了这个页面就算大胜利：

差不多了，数据库账号这些自己按需分配，你可以给不同用户分配不同的权限，某些框架会使用用户来完成对数据库的操作。
