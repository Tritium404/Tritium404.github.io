Linux 下的包管理

对于 Debian 系

dpkg 命令

由于 dpkg命令现在很少用，现在只列一个简表，更多命令可以通过 dpkg --help 查询。

参数 示例 补全 操作
-i dpkg -i package.deb –install 安装一个 deb 包
-l dpkg -l package –list 列出该包的简要信息（版本号，架构，描述）
-L dpkg -L package –listfiles 列出与该包相关的路径
-r dpkg -r package –remove 仅移除包，不删除配置文件
-R dpkg -R / –recursive 递归参数
-p dpkg -p package –print-avail 显示可用的版本细节
-P dpkg -P package –purge 完全移除一个包
-s dpkg -s package –status 展示此包的细节及状态
-S dpkg -S pattern –search 检索包归属的文件
-C dpkg -C package –audit 检查是否有损坏的包
-A dpkg –A package.deb –record-avail 记录可用包
-V dpkg -V package –verify 验证包的完整性
-？ dpkg -? –help 显示帮助

关于 dpkg 状态的修复

在执行各种包管理命令时，dpkg 存在一个状态文件，来记录软件是否被安装；版本号是多少；配置文件存在否，以及它在哪里等等。

apt-get 命令

apt-get 命令更加简单易用，可以看作是基于 dpkg 命令的实现。

参数 描述
update 更新软件包列表至本地
upgrade 更新软件，但若依赖文件出现相互依存，或者升级会破坏其他的文件依赖时候停止升级
install 安装
reinstall 重新安装
remove 卸载软件，但保留其配置
autoremove 移除软件及其依赖（可能会让其他需要此依赖的软件出错）
purge 卸载软件并清除其配置
dist-upgrade 更新软件，如果有依赖性问题，尝试解决和移除。会造成文件依赖的破坏风险。
dselect-upgrade 使用 dselect 更新（大概相当于 apt-get update 与 apt-get upgrade 连招）
build-dep 安装编译某个软件包的环境
satisfy 安装依赖
clean 无差别清理安装包
autoclean 清理安装包，但会保留已安装软件的软件包
check 检查是否有损坏的依赖
source 获取某个软件的源码
download 下载二进制包
changelog 更新日志
忽略更新
命令 描述
apt-mark hold packagename 忽略某个包的更新
apt-mark unhold packagename 取消忽略
次级参数
参数 描述
-h 帮助文件
-q 输出到日志 - 无进展指示
-qq 不输出信息，错误除外
-d 仅下载 - 不安装或解压归档文件
-s 不实际安装。模拟执行命令
-y 在需要确认的场景中回应 yes
-f 尝试修正系统依赖损坏处
-m 如果归档无法定位，尝试继续
-u 同时显示更新软件包的列表
-b 获取源码包后编译
-v 显示详细的版本号
-c=? 阅读此配置文件
-o=? 设置自定的配置选项，如 -o dir::cache=/tmp
